####code explanation This heart of the thing is the while loop that does the divide and conquer. `l` can never equal `r`. Once `l` and `r` are 1 apart, `m` will go back to the same place it was when they were 2 apart and the while loop will never return. That's because the modulus `Math.floor((r+l)/2)` is the same for two apart as it is for one apart: `Math.floor((2+4)/2)==Math.floor((3+4)/2)`. The while loop cannot exit at `r-l< 1` which means it will have two elements after the last divide and conquer. If it is not in the first you have to check the second function findLastDup(a,v){ var l=0, r=a.length-1, m=Math.floor((r+l)/2) while (true){ if (v==a[m]){ return theIdxs(m) }else if(v==a[m+1]){ return theIdxs(m+1) }else if (v>=a[m]){ l=m; }else { r=m; } m=Math.floor((r+l)/2) if (r-l < 2){ return -1 } } } Once you have found one value, you can't be sure if you found the first repeated value, the last repeated value or somewhere in the middle. `theIdx()` checks that. First it checks backwoods while prior values are the same and careful not to back off the beginning of the array. Then it resets the search point and checks the other side. It returns an array of the indexes of all the repeated elements then finds the max applying Math.max to the array`Math.max.apply(null,ret)` function theIdxs(m){ var to = m var ret =[] ret[0]=m ti=to //check both ways for more or the same while (ti-1<=0 && a[ti]==a[ti-1]){ ti-- ret.push(ti) } ti=to while (ti+1< a.length-1 && a[ti]==a[ti+1]){ ti++ ret.push(ti) } var max = Math.max.apply(null,ret) return max }
